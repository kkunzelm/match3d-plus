What was the problem: The entire point-pair picking pipeline was already
  implemented — DepthImageView landmark mode, the state machine in
  MatchingControlPanel, and CoarseRegistration::fromPoints() — but the three
  Match menu entries (From points, From COM..., Clear parameters) were
  permanently disabled with no connections.

  What was fixed:
  - onFromPoints(), onFromCOM(), onClear() moved from private slots to public 
  slots in MatchingControlPanel
  - The three menu items are now enabled and wired: they open/raise the Matching
   Control Panel (creating it if needed) and immediately invoke the action
   
  How to pick point pairs now:
  1. Open an image window and go to Match → From points
  2. The Matching Control Panel opens — select the target image in the list
  3. Click corresponding points in the data image (numbered yellow circles
  appear); press Enter or double-click when done (minimum 3 points)
  4. The panel switches automatically to the target image — click the matching
  points in the same order
  5. After the last target point (count matches data), the transform is computed
   automatically and populated into the parameter spinboxes
  6. Use Match → Match (or the "Match" button) to run ICP from that starting
  position
